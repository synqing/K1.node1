# Metadata

Taxonomies, schema, registries for design documentation.
- Keep component registry aligned with code.
- Use frontmatter in component docs for measdtadata.

## Management
- Tagging facets: `area`, `component`, `capability`, `status`, `version`, `accessibility`, `performance`.
- Categorization: registry and controlled vocabularies live here.
- Versioning: use semantic versioning for specs and tokens; track changes centrally.
- Schema: see `design/05-metadata/METADATA_SCHEMA.md` for machine-readable schema and examples.
