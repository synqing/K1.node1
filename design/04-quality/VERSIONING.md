# Versioning and Change Tracking

Principles:
- Semantic versioning for component specs and design tokens.
- Changelogs per component and a central `design/04-quality/CHANGELOG.md`.

Protocol:
- Increment `minor` for new variants or non-breaking styling changes.
- Increment `major` for token changes that affect contrast or spacing.
- Increment `patch` for fixes.

Traceability:
- Each change references issue IDs and links to code diffs.
- Include metadata updates (`status`, `updated_at`).
- Update central changelog at `design/04-quality/CHANGELOG.md`.
